:::danger

In the above example, ff we assign `x`, and `y` both to be `2147483647` (max
`i32`), the result will overflow `i32` range. Unfortunately, such errors are not
caught compile time.

Check out helper methods: [wrapping_add](https://doc.rust-lang.org/std/primitive.i32.html#method.wrapping_add), [saturating_add](https://doc.rust-lang.org/std/primitive.i32.html#method.saturating_add), [overflowing_add](https://doc.rust-lang.org/std/primitive.i32.html#method.overflowing_add), [checked_add](https://doc.rust-lang.org/std/primitive.i32.html#method.checked_add).

:::
